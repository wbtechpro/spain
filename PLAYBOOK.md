# Как я работаю с этим проектом через Клода

Я не хожу на сервер и не запускаю команды руками. Всё делается через диалог
с Claude Code. Эта памятка — мои триггер-фразы и правила.

**`CLAUDE.md`** — для Клода (читает автоматически).
**`PLAYBOOK.md`** — для меня (этот файл, чтобы не забыть как что просить).

---

## Как открыть новую сессию

1. Открываю Terminal / iTerm.
2. `cd ~/Desktop/spain-map` (или клик по каталогу в Claude Code).
3. `claude` — запускается новая сессия.
4. `CLAUDE.md` подтянется автоматически — Клод уже знает про ветки, автодеплой и прочее.

**Когда открывать новую сессию:**
- Начал принципиально другую задачу (была фича — теперь баг).
- Прошёл день, контекст устарел.
- Сессия затупила / зависла / стала вести себя странно.

**Когда продолжить старую:**
- Та же задача, просто отвлёкся.
- Нужен тот же контекст для ревью / уточнения.

## Типовые задачи — что писать

### Добавить слой на карту

> «Добавь слой X: источник такой-то, цвет #xxxxxx, иконка 🦊.
>  Колонки: name, lat, lng, website. Расскажи как проверить.»

Клод сам создаст `scripts/fetch_X.py`, дополнит `server/config.py`,
добавит строку в панель, прогонит через PR → автодеплой.

### Обновить данные

> «Обнови данные по аренде (SERPAVI). Ветка `data/refresh-rent`.»

Или короче:
> «Перезапусти `fetch_rent.py` и выкати.»

### Починить баг в UI

> «На проде счётчик schools не обновляется при зуме. Посмотри, почему.»

Клод откроет DevTools-логи через preview или `curl`, найдёт и починит.
Я только проверяю результат в браузере.

### Посмотреть что на проде

> «Проверь что https://dubson.org живой и `/api/counts` отвечает.»
> «Покажи логи бэкенда за последний час.»
> «Покажи последний deploy в GitHub Actions.»

### Откатить

> «Откати последний merge — сломался кластеринг.»

Клод сделает revert-PR → автодеплой → прод возвращается за минуту.

### Добавить что-то быстрое на прод (тайпо, текст)

> «Поправь заголовок сайдбара на „Карта Испании для Фёдора" и выкати.»

Даже для однострочника — через feature-ветку и PR. Автомат.

## Что я НЕ делаю руками

| Задача | Кто делает |
|---|---|
| `git` команды | Клод |
| `ssh root@...` на VPS | Клод |
| `systemctl` / `journalctl` | Клод (через ssh) |
| Запуск fetch-скриптов | Клод |
| Деплой | GitHub Actions после merge |
| Смотреть логи, дебажить API | Клод через `curl` / preview |

Я: пишу промпты, смотрю результат на https://dubson.org, апрувлю направление.

## Параллельные задачи

Если хочу, чтобы Клод начал одну задачу, а я параллельно дал ему другую:

> «Запусти пересборку данных по школам в worktree `../spain-data`,
>  а тут давай править UI-фикс.»

Клод создаст отдельный каталог через `git worktree` — не смешается.

## Типовые проблемы и триггер-фразы

| Проблема | Что писать Клоду |
|---|---|
| Прод сломался | «Прод отдаёт 500. Проверь логи, откати если нужно.» |
| Что-то не видно в браузере | «Запусти preview, покажи console + network.» |
| Сомневаюсь в промежуточном результате | «Покажи git diff перед коммитом.» |
| Не знаю как назвать ветку | «Сам подбери имя по Conventional Commits.» |
| Две сессии столкнулись | «Ветка отстала от main, заребейзь.» |
| Секреты / ключи | «Сгенери, положи в GitHub Secrets, локально удали.» |

## Хорошие привычки

- **Маленькие задачи — один PR — один merge.** Проще откатить, проще ревьюить.
- **Промпт в одну сторону.** «Сделай X» лучше, чем «сделай X, Y, Z и потом Q,
  если останется время». Второе провоцирует галлюцинации приоритетов.
- **Если не уверен — сначала «расскажи план, не делай».** Клод распишет шаги,
  я корректирую, потом говорю «поехали».
- **На сомнительных изменениях — «сначала в PR, не мержь».** Могу посмотреть
  diff на github.com перед автодеплоем.

## Если Клод не знает контекста

> «Прочитай `CLAUDE.md` и `PLAYBOOK.md`, потом продолжим.»

Обычно не нужно — он сам читает при старте. Но если сессия долгая, напоминание помогает.
